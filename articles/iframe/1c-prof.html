Объекты конфигурации

Если реквизиту указать тип "Дата", то в нем может содержаться:

  • Дата
  • Время
  • Дата+Время
  • Дата или Дата+Время
  • Дата или Время или Дата+Время

Массив:

  • Является самостоятельным типом данных
  • Относится к универсальным коллекциям значений
  • Верны оба высказывания

В общих модулях ...

  • Нельзя описывать переменные
  • Нельзя описывать переменные, процедуры и функции без ключевого слова "Экспорт"
  • Нет ограничений на описание переменных, процедур, функций

Модуль объекта существует у объектов вида:

  • Планы видов характеристик
  • Документы
  • Отчеты, обработки
  • Регистры
  • У всех перечисленных объектов, кроме регистров

Ключ может быть произвольного типа:

  • У структуры
  • У соответствия
  • Ни у одного из вышеперечисленных

Функциональная опция может хранить свое значение

  • В константе
  • В реквизите справочника
  • В ресурсе регистра сведений
  • Во всех вышеуказанных объектах

У предопределенного элемента справочника пользователь не может

  • Изменить код
  • Изменить наименование
  • Изменить наименование и код
  • Нет правильного ответа

Справочник может быть подчинен:

  • Нескольким справочникам и планам видов характеристик
  • Только справочникам
  • Только планам видов характеристик
  • Только одному справочнику или одному плану видов характеристик

При добавлении в конфигураторе объекта вида "Справочник" в допустимых типах для реквизита объекта …

  • Появляется один дополнительный тип значений
  • Появляются два дополнительных типа значений
  • Появляется большее количество дополнительных типов значений
  • Дополнительных типов для реквизитов объектов не возникает

Для обращения к предопределенному элементу справочника необходимо:

  • Найти его по неизменяемому коду: Справочники.ИмяСправочника.НайтиПоКоду(Код)
  • Найти его по неизменяемому наименованию: Справочники.ИмяСправочника.НайтиПоНаименованию(Наименование)
  • Найти его по имени: Справочники.ИмяСправочника.ИмяЭлемента
  • Програмно найти предопределенный элемент нельзя, пользователь просто его не может удалить и пометить на удаление

Процедуры/функции, описанные в общем модуле, работают:

  • На стороне клиента
  • На стороне сервера
  • В зависимости от директивы предваряющей описание процедуры/функции либо на клиенте, либо на сервере
  • В зависимости от флагов "Клиент", "Сервер" и директив предваряющих описание процедуры/функции (в случае отметки обоих флагов) либо на клиенте, либо на сервере

Реквизит формы:

  • Доступен в контексте этой формы
  • Доступен в модуле соответствующего объекта
  • Верны оба высказывания

Процедуры/функции, описанные в модуле объекта, работают:

  • На стороне сервера
  • В зависимости от директивы предваряющей описание процедуры/функции либо на клиенте, либо на сервере

Интеграции

Изменения регистрируемые планом обмена регистрируются:

  • 1. Вне зависимости от количества определенных в нем узлов
  • 2. Для каждого определенного на момент регистрации узла
  • 3. Нет правильного утверждения

Конфигурация "Конвертация данных" используется:

  • 1. Для разработки и модификации правил обмена данных
  • 2. Для разработки и модификации правил регистрации объектов
  • 3. Все варианты верны
  • 4. Нет верного варианта

При использовании механизма распределенных баз данных изменения в конфигурации:

  • 1. Выполняются в каждой информационной базе самостоятельно
  • 2. Передаются посредством файла формата cf
  • 3. Изменения включаются в стандартные сообщения обмена
  • 4. После распределения информационной базы изменение конфигурации запрещено

У плана обмена может быть следующее количество предопределенных узлов

  • 1. Ни одного
  • 2. Всегда один
  • 3. Любое количество

Роли

Тест

Для запрета режима непосредственного удаления объектов пользователем необходимо:

  • 1. В свойствах конфигурации снять флаг "Разрешить непосредственное  удаление объектов"
  • 2. При определении роли для соответствующих объектов снять отметку у флага "Удаление"
  • 3. При определении роли для соответствующих объектов снять отметку у флага "Интерактивное удаление"
  • 4. В свойствах роли снять отметку у флага "Разрешить удаление объектов"
  • 5. В свойствах роли снять отметку у флага "Разрешить интерактивное удаление объектов"

Роль, как объект конфигурации, отражает:

  • 1. Статус и сферу ответственности ее носителя как сотрудника компании
  • 2. Совокупность доступных определенному пользователю объектов в среде "1С:Предприятие"
  • 3. Совокупность прав действий в среде "1С:Предприятие" с возможностью присвоения конкретному пользователю
  • 4. Набор параметров пользователя

Для запрета режима непосредственного удаления предопределенных элементов справочника пользователем необходимо...

  • 1. В свойствах конфигуратора снять флаг «Разрешить непосредственное удаление предопределенных»
  • 2. При определении роли для соответствующих объектов снять отметку у флага «Интерактивное удаление предопределенных»
  • 3. При определении роли для соответствующих объектов снять отметку у флага «Интерактивное удаление»
  • 4. В свойствах роли снять отметку у флага «Разрешить интерактивное удаление предопределенных»
  • 5. В свойствах роли снять отметку у флага «Разрешить интерактивное удаление объектов»
  • 6. Возможности удаления предопределенных элементов в системе нет

При попытке выполнить запрос с текстом «Выбрать * Из Справочник.Номенклатура», в случае если на записи справочника были определены ограничения на чтение (в соответствующей роли) произойдет следующее:

  • 1. Будут получены все данные
  • 2. Будут получены данные только из разрешенных записей
  • 3. Произойдет ошибка

При настройке ограничений прав доступа (в ролях), при определении шаблонов:

  • 1. в них можно определить один параметр, значение которого будет передаваться при указании шаблона в конкретном ограничении доступа к данным
  • 2. в них можно определить любое количество параметров, значение которых будет передаваться при указании шаблона в конкретном ограничении доступа к данным
  • 3. параметры в шаблонах использовать нельзя
  • 4. Определение шаблонов и заключается в описании параметров

При настройке ограничения прав доступа на уровне записей и полей базы данных значения параметров для условий определяются

  • 1. как переменные, определенные в модуле приложения
  • 2. как переменные, определенные в модуле приложения с ключевым словом Экспорт
  • 3. как параметры сеанса
  • 4. как константы
  • 5. как переменные определяемые в соответствующем обработчике события
