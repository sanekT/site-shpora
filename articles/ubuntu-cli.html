Работа с браузером

Чтобы запустить из терминала браузер firefox просто пишем команды: :~$ firefox или chromium

Чтобы запустить файл index.html в браузере, нужно зайти в папку с файлом и ввести команду : ~$ firefox index.html

x-www-browser - определяем браузер по умолчанию

Устанавливаем node js

sudo apt-get install nodejs

sudo apt install nodejs-legacy

Выполняем эти 2 команды. Затем можно проверить версию установленного node:

node -v

Затем проверяем установлен ли npm командой:

npm -v

Если не усиановлен, то устанавливаем командой:

sudo apt install npm

Есть один косяк - установится версия node 4.2.4, а это уже старая версия. Поэтому удаляем node js:

sudo apt-get remove nodejs

При удалении может возникнуть проблемма, система не захочет удалять, порывшись в интернете я обнаружил, что не удаляется из за того, что в системе запущен где то ещё один или несколько процессов apt-get, их нужно закрыть. Файловая система Ubuntu

Что из себя представляет корневая файловая система Linux? Прежде, чем приступить к командам навигации в терминале, поговорим о директориях в Linux. Здесь нет дисков C:, D: и прочих. Неважно, на сколько разделов вы разобьёте ваш диск, в системе будет:

/ - корневая директория

/home - хранит каталоги пользователей, в которых можно, как правило, выполнять большинство действий без пароля администратора. Ваш домашний каталог находится по адресу: /home/Ваше_Имя_пользователя.

/mnt и /media - в эти каталоги монтируются другие физические диски, флешки и прочие носители информации.

/media - сюда монтируются диски, флешки, то есть, в этом каталоге создается папка с названием вашего электронного носителя.

Команды навигации в терминале

Ctrl + Alt + T - открыть терминал

pwd - Когда вы открываете терминал в Ubuntu, то вы находитесь в своем домашнем каталоге, при вводе команды pwd, нам покажет текущую директорию.

Значок ~ (тильда) - означает домашний каталог текущего пользователя.

Для навигации в терминале используется команда "cd" (англ. change directory - изменить каталог). Сначала пишется команда cd, а затем путь, куда нужно перейти. Вот так выглядит переход в каталог /home: cd /home

cd .. - Перейти в директорию уровнем выше

cd ../.. - Перейти в директорию двумя уровнями выше

cd - Перейти в домашнюю директорию вашего пользователя

cd ~user_name - Перейти в домашнюю директорию пользователя user_name

cd - - Перейти в директорию, в которой находились до перехода в текущую директорию

Следующая команда, используемая в навигации - "ls" (сокращение от англ. list). Если ввести в терминале "ls", то отобразаится содержимое текущей директории:

ls -F - Отобразить содержимое текущей директории с добавлением к именам символов, характеризующих их тип

ls -l - Показать детализированное представление файлов и директорий в текущей директории

ls -a - Показать скрытые файлы и директории в текущей директории

ls -la - Ключи можно комбинировать вместе

ls --help - Кроме того, у команды ls очень много других ключей, которые можно посмотреть, выполнив команду

Автодополение названий директорий в терминале. Когда вы вводите команду cd, вам необязательно вводить полностью имя директории, достаточно нажать на клавиатуре клавишу "TAB", произойдет автодополнение названия каталога. Если имя не заполнилось, значит, есть еще один каталог с таким названием. Нажав дважды клавишу "TAB", выведется список с каталогами, которые имеют в своём имени начальные буквы, которые вы ввели. Если список огромнейший, то для отмены его просмотра нажмите клавишу N. Это касается не только "cd", но и других команд. Например, когда вы хотите отредактировать какой-то файл. Вы выполнили команду ls, увидели, что у файла имя состоит из 30 знаков, то достаточно ввести первые буквы его имени и нажать клавишу "TAB" и произойдет автодополнение. Это основные команды навигации в терминале Linux.

Команды создания, удаления, копирования, перемещения, переименования файлов и директорий

whoami - узнать от имени какого пользователя мы сейчас активны

sudo apt-get update - обновить скопированный на компьютер список пакетов репозитория APT

sudo apt-get install aptitude - установить такой же менеджер пакетов как и apt-get, но более функциональный и удобный

sudo aptitude safe-upgrade - эта команнда говорить - "всё что сейчас на локальной машине установленно, сходи на репозиторий посмотри свежии версии какие есть и обнови, но так чтобы ничего не сломалось"

переменные окружения

Эти переменные задаются автоматически убунтой

echo $USER - выведет текущего пользователя

echo $HOME - выводит домашню папку пользователя

echo $PWD - выводит текущую директорию

cp file.txt file2.txt - копирует файл

cp -r papka papka2 - копирует папку, опция -r означает копировать рекурсивно, т.е всё что в папке тоже копируется

Ctrl + H - если находимся в проводнике, то показывает скрытые файлы

Ctrl + L - очищает экран терминала как и команда clear

В домашней директории есть 3 файла: .bashrc, .bashrc_profile, .profile - это файлы настройки нашей оболочки, в нашем случае нашего bash-оболочки. Команды внутри этих файлов выполняются перед тем как запустится оболочка. РАзница между этими файлами в том, когда они выполняются. .profile и bash_profile выполняютс при логине в систему, а .bashrc выполняется каждый раз, когда создаётся новая сессия нашей оболочки bash. Это что то вроде автозагрузки windows. Мы можем например открыть файл .bashrc и в начале записать новую переменную и тут же вызвать её:

HELLO="Hello"
echo $HELLO

При запуске терминала нам выведется вначале значение этой переменной

php -m - показывает установленные php модули

php -S localhost:5555 - запускает встроенный сервер в php

права доступа

sudo chmod -R 755 /path/to/directory - изменить права доступа у владельца папки, например вледелец root. -R означает, что изменения пойдут рекурсивно внутрьыва папки. Число 755 это:

7 - rwx - читать, писать, выполнять
6 - rw- - читаь, писать
5 - r-x - читать, выполнять
4 - r-- - читать
3 - -wx - писать, выполнять
2 - -w- - писать
1 - --x - выполнять
0 - --- - ничего нельзя делать

Что означают цифры применения прав 755? Первая - для пользователя, вторая - для группы, третья - права для всех остальных

Для смены владельца файлов/каталогов используем chown:

$ chown -R (пользователь):users /home/user/ваша папка/

Права доступа для Apache

Я настроил общую папку для Ubutnu и Windows. В Ubuntu сделал ссылку для общей папке в директорию /var/www/ :

$ ln -s /media/sf_yii2/ /var/www/

Теперь у меня есть такой путь /var/www/sf_yii2/, в этом пути уже общие файлы, в этой папке создадим папку для сайта и настроим её в apache, укажем к ней путь и добавим в hosts, но apache в итоге скажет что не имеет доступа к этой папке, даём доступ к папке добавляя к apache ещё одну группу, которая имеет доступ к папке sf_yii2, это группа vboxsf:

$ sudo usermod -a -G vboxsf www-data

$ ifconfig - выведет гифу о компе, а также покажет ip адрес
