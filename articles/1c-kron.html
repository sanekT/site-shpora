Галочку "Глобальный" никогда не ставим. Далее создаём подписку на событие, например будем подключаться к событию "ПереЗаписью" документа поступления. Затем нужно выбрать "Событие" из модуля этого объекта, к которому хотим присоединиться, например, "ОбработкаПроведения". Ну и выбираем наш обработчик в общем модуле, который будет вызван после обработчика "ОбработкаПроведения". При обновлении типовой система не трогает подписки и общие модули с ними связанные. Составной тип можно указать для свойства "Источник", но тогда может уменьшится количество доступных событий в свойстве "События", потому что не у всех одинаковых событий разных объектов(документов, справочников) есть одинаковые параметры. Если парамтеры не совпадают, а событие одно и то же, то такое событие нельзя будет выбрать.

БухУчет

  • День 1 часть 1
  • 00:00:00 - начинаем бух учет.

    Оперативный учет это учет, который решает оперативно какие то локальные задачи, где нужно получить текущие(оперативные) данные для управления предприятием. Например, для торговой организации самые опреативные данные это товары на складах, взаиморасчеты с контрагентами. Для производства это уже производственный учет. Т.е. эти учеты охватывают организационную и финансовую деятельность в локальных областях предприятия.
    Бух учет охватывает уже всю финансовую и хозяйственную деятельность предприятия. Далее по картинке пробежался и рассказал про остальные области бух учета. Эта схема не есть самый правильный способ разделения бух учета, могут быть варианты.
