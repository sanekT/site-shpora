Зачем нужны индексы в 1С?

Для чего нужны индексы в 1С?

Индексы нужны для оптимизации операций поиска данных. Они позволяют системе быстрее находить необходимые записи в таблицах, что особенно важно при работе с большими объёмами данных. Это приводит к уменьшению времени выполнения запроса.

Если привести понятную аналогию из жизни, то индексы это как содержание книги.

Если содержания нет, то нужно будет листать книгу с самой первой страницы до нужной нам. Так и в 1С, если индексов нет, то система будет перебирать все записи в поисках нужной.

Преимущества:

Снижение времени поиска данных. Поиск «перебором»: 1 000 000 операций. Поиск по B-дереву: 14 операций

Недостатки:

- Операции(вставка, изменение, удаление) изменения данных таблицы требую актуализации всех индексов.

- Индекс занимает место.

- Индексы нужно обслуживать (фрагментация индексов на диске).

Поэтому нужно с умом подходить к созданию индексов. Не создавать их беспорядочно, а только там где это действительно нужно.

Какие индексы бывают:

Кластерные:

  • Записи БД сортируются по ключу кластерного индекса. Кластерный индекс это не отдельная сущность, механизм сортировки данных
  • На уровне листьев собственно данные
  • В 1С есть почти у всех таблиц

Пример:

Есть таблица элементов справочника. Индексов нет и данные записаны как попало, так называемая "куча". Система сама создала индекс по ссылке и записи в таблице отсортировались по ссылке. И рядом с таблицей или над таблицей построилось В-дерево. При поиске какой-то записи по ссылке в ключ индекса выбирается эта ссылка и по промежуточным веткам система идёт к нужной записи.

Не кластреный

  • Полностью отдельная структура. Может быть несколько на одну таблицу
  • На уровне листьев ключ кластерного индекса
  • В 1С создаётся автоматически, а часть при установке «Индексировать»
  • Не кластерный индекс менее производителен чем кластерный
